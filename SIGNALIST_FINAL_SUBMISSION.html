ABSTRACT

The modern financial era is dominated by an overwhelming volume of real-time information, often resulting in "Analysis Paralysis" for retail investors. This project, "Signalist," introduces an advanced AI-driven stock intelligence platform designed to bridge the gap between raw data and actionable insights. Built using the Next.js 15 framework, Signalist provides a lightning-fast "Bento Grid" interface for monitoring global equity markets.

The core innovation is the integration of Google Gemini Flash for automated market sentiment analysis and daily news digests. By leveraging an event-driven architecture powered by Inngest, Signalist orchestrates background workflows that fetch, summarize, and deliver personalized market intelligence. The project demonstrates a scalable full-stack serverless implementation that democratizes high-frequency financial insights, providing users with a proactive, intelligence-first wealth management tool.

1. INTRODUCTION

1.1 Project Overview

Signalist is an intelligence-native financial analytics system. It moves beyond traditional price-tracking by acting as an automated research analyst for the retail investor. The system consolidates real-time price action from professional data providers with contextual summaries generated by Artificial Intelligence.

1.2 Motivation

The motivation for this project is the massive information gap between institutional traders and retail investors. Institutional players have access to high-priced analysts; Signalist uses Generative AI to provide a similar level of synthesis to the public for free.

1.3 Objectives

Our objectives are threefold: First, to minimize market research time. Second, to provide a zero-latency dashboard experience. Third, to automate the delivery of market intelligence via proactive email digests.

2. LITERATURE SURVEY

Quantitative finance has evolved from simple moving average cross-overs to complex deep-learning models. However, the rise of Large Language Models (LLMs) represents a "second revolution." Unlike traditional ML models that predict price numbers, LLMs understand the "narrative" behind the news—legal battles, product launches, and geopolitical shifts. Signalist's literature survey highlights how LLMs are currently outperforming traditional sentiment analyzers in the financial domain.

3. SYSTEM ANALYSIS

The system was analyzed for feasibility across Technical, Economic, and Operational dimensions. Next.js was selected due to its superior Static Site Generation (SSG) and Server-Side Rendering (SSR) capabilities. Hardware requirements were kept minimal to ensure the application remains accessible on mobile and lower-end desktop devices.

4. SYSTEM DESIGN

The project follows a Micro-services inspired Serverless Architecture. Each core functionality—Search, Tracking, AI Summarization—is decoupled into discrete serverless functions. This ensures 99.9% availability and high fault tolerance. Database modeling in MongoDB uses a normalized approach for users and symbols to maximize query efficiency.

5. IMPLEMENTATION

The implementation phase focused on the Inngest orchestration layer. By using durable execution, we ensure that if an AI call fails, the system automatically retries without user intervention. The frontend utilizes Tailwind CSS for a professional dark-mode 'Bento Box' aesthetic.

'use server'; const FINNHUB_KEY = process.env.NEXT_PUBLIC_FINNHUB_API_KEY; export async function searchSymbols(query: string) { if(!query) return []; try { const res = await fetch(`https://finnhub.io/api/v1/search?q=${query}&token=${FINNHUB_KEY}`, { next: { revalidate: 3600 } }); const data = await res.json(); return data.result || []; } catch (e) { return []; } } export async function getMarketNews(symbols: string[]) { // Logic to aggregate news for specific symbols const promises = symbols.map(s => fetch(`https://finnhub.io/api/v1/company-news?symbol=${s}&token=${FINNHUB_KEY}`)); const results = await Promise.all(promises); return results.map(r => r.json()); }
